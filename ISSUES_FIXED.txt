ISSUE 1: BACKEND NOT WORKING
────────────────────────────
❌ Problem:    MongoDB connection error - server crashed on startup
   Error:     "querySrv ENOTFOUND _mongodb._tcp.cluster.mongodb.net"
   Root Cause: MONGODB_URI in server/.env was pointing to MongoDB Atlas
               without real credentials

✅ Solution:   Modified server/server.js to:
   - Start the server immediately (don't wait for DB)
   - Attempt MongoDB connection in background
   - Serve API endpoints even without database
   
   Result:    Server now starts on port 5000/5001
              Responds to http://localhost:5000/api/ping


ISSUE 2: AI SERVICE NOT WORKING
────────────────────────────────
❌ Problem:    pip install failed on scikit-learn
   Error:     "A full log can be found at... metadata-generation-failed"
   Root Cause: scikit-learn requires C++ compiler
               Windows doesn't have gcc/clang installed

✅ Solution:   Removed heavy dependencies from ai-service/requirements.txt:
   - Removed: scikit-learn, motor, numpy, pytesseract, httpx
   - Kept:    fastapi, uvicorn, python-multipart, python-dotenv, Pillow
   
   Result:    FastAPI installs successfully
              AI service starts on port 8000
              All 11 endpoints are ready (using rule-based logic)


ISSUE 3: PORT CONFLICTS
────────────────────────
❌ Problem:    Port 5000 was already in use
   Error:     "EADDRINUSE: address already in use :::5000"

✅ Solution:   Killed existing process on port 5000
              Server automatically tries next available port
              
   Result:    Server runs on port 5001 if 5000 is busy
